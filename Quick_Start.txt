.US Locality DNS Discovery Tool — Quick Start
==============================================

Goal: scan known 3rd-level .US domains from our system and look for child DNS
names beneath them that are not already known in the system.

Recommended enriched CSV (six columns only):

domain,delegated_manager,known_fourth_level_domains,known_fifth_level_domains,fourth_level_count,fifth_level_count
state.wv.us,Example Manager,,,0,0
auburn.in.us,Mirage Computers,ci.auburn.in.us,,1,0

Column guide:
* domain — known 3rd-level .US domain to scan (for example state.wv.us)
* delegated_manager — manager/operator context
* known_fourth_level_domains — 4th-level domains already in the system
* known_fifth_level_domains — 5th-level domains already in the system
* fourth_level_count / fifth_level_count — counts from the system (context)

Leave known child fields blank if none are known. Do not include duplicate
domain columns. Extra spreadsheet columns (zone, locality_label, notes, etc.)
are not required.

1. Run USLocalityDNSDiscovery.exe
   (Copy to a local folder if Windows SmartScreen blocks a network share.)

2. Select the simplified CSV, or TXT / simple CSV for domain-only lists.

3. Choose Output Folder
   Default is output\ beside the EXE. Use Browse to pick another writable folder.

4. Select scan profile
   Start with Light Evidence for a 10–25 domain sample (default).
   Keep Attempt AXFR off for first-pass Light Evidence scans unless you need it.

5. Review Preflight Summary
   Look for "Preferred input format detected" when using the six-column CSV.
   Confirm selected domain column and first domains shown.

6. Run Scan
   Watch the phase/progress text. Cancel Scan stays available during the run.

7. Export XLSX (recommended)
   CSV, JSON, and All formats also write to the selected Output Folder.

8. Open Evidence Review first
   Rows are sorted by evidence strength (Strong → Moderate → Limited → etc.).
   Column headers are human-readable (for example "Base domain", "Why").

9. Read the Why column; open How to Read for definitions
   - Strong: new delegated child zone (NS/SOA) not in system input
   - Moderate: new meaningful child name not in system input
   - Limited: generic/technical hostname or alias to known domain
   - Validation only: known domain confirmed in DNS (not a new discovery)

10. Use Verification guidance for optional independent dig checks.

Important:
* zone and second_level_domain are optional — workbook fields may be blank.
* www / autodiscover / mail are included as DNS names but usually limited value.
* No child names discovered does not prove absence.
* A few good examples may be enough.

Manual operator checklist (packaged EXE):
* Browse — select domain list CSV/TXT
* Output Folder → Browse — pick writable report folder (or use default beside EXE)
* Output Folder → Open Folder — open selected folder after export
* Scan Profile — Light Evidence (default) / Normal / Deep Targeted
* Run Scan — watch phase/progress text
* Cancel Scan — stop at next safe checkpoint if needed
* Export Results — XLSX recommended; CSV/JSON/All also available

See README.md for full evidence model and handoff details.
